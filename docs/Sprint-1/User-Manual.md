#  คู่มือการใช้งานระบบรายงาน (Report System User Manual)

> โปรเจค **Painamnae** — ระบบแชร์การเดินทาง

---

## สารบัญ

1. [ระบบรายงานเหตุการณ์สำหรับคนขับ (PBL11)](#1-ระบบรายงานเหตุการณ์สำหรับคนขับ-pbl11)
2. [ระบบรายงานพฤติกรรมคนขับสำหรับผู้โดยสาร (PBL13)](#2-ระบบรายงานพฤติกรรมคนขับสำหรับผู้โดยสาร-pbl13)

---

## ภาพรวมระบบ

ระบบ Painamnae Backend เป็น API Server พัฒนาด้วย **Node.js + Express** ร่วมกับ **Prisma ORM + PostgreSQL** ทำหน้าที่จัดการข้อมูลผู้ใช้ การเดินทาง และระบบ Incident Report

### ฟีเจอร์หลักของระบบรายงาน

| ฟีเจอร์        | รายละเอียด                                               |
| -------------- | -------------------------------------------------------- |
|  สร้างรายงาน | คนขับและผู้โดยสารสามารถส่งรายงานเหตุการณ์ได้             |
|  แนบหลักฐาน  | รองรับการอัปโหลดรูปภาพและวิดีโอประกอบ                    |
|  ติดตามสถานะ | ดูสถานะเคสได้แบบ real-time (รอดำเนินการ / ดำเนินการแล้ว) |

---

## 1. ระบบรายงานเหตุการณ์สำหรับคนขับ (PBL11)

> **User Story:** As a driver, I want to report incidents to the admin and get the update on the filed case.

ผู้ขับขี่สามารถรายงานเหตุการณ์หรือพฤติกรรมที่ไม่เหมาะสมที่เกิดขึ้นระหว่างการให้บริการไปยังผู้ดูแลระบบ เพื่อให้ตรวจสอบและดำเนินการแก้ไขปัญหาได้อย่างรวดเร็ว โดยระบบจะบันทึกรายงานแต่ละรายการในรูปแบบเคส (Case) พร้อมหมายเลขอ้างอิง รายละเอียดเหตุการณ์ หลักฐานประกอบ และสถานะการดำเนินงาน เพื่อให้สามารถติดตามความคืบหน้าและตรวจสอบย้อนหลังได้อย่างเป็นระบบ

## 2. ระบบรายงานพฤติกรรมคนขับสำหรับผู้โดยสาร (PBL13)

> **User Story:** As a passenger, I want to report the driver behavior to the admin and get the update on the filed case.

ผู้โดยสารสามารถรายงานพฤติกรรมของคนขับที่ไม่เหมาะสมหรือเหตุการณ์ที่เกิดขึ้นระหว่างการเดินทางไปยังผู้ดูแลระบบ เพื่อให้ตรวจสอบและดำเนินการแก้ไขปัญหาได้อย่างรวดเร็ว โดยระบบจะบันทึกรายงานแต่ละรายการเป็นเคส (Case) พร้อมหมายเลขอ้างอิง รายละเอียดเหตุการณ์ หลักฐาน และสถานะ เพื่อให้ติดตามความคืบหน้าได้

### ฟีเจอร์หลัก

|  สร้างรายงาน | ผู้โดยสารรายงานพฤติกรรมคนขับได้ |
|  แนบหลักฐาน  | รองรับรูปภาพและวิดีโอ           |
